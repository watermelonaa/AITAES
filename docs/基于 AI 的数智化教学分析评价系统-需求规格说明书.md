## 1. 引言 

### 1.1 编制目的 

本文档旨在明确“高校课程教学智能分析系统”的功能需求、性能指标及接口规范，为后续的系统设计、编码实现、测试验收及项目交付提供统一、无歧义的依据。本文档的预期读者包括项目团队成员（开发、测试、文档）、指导教师及答辩评审专家。

### 1.2 范围 

本系统定位于 **以单门课程为核心、教师为主导、AI 为驱动力的教学辅助工具** 。系统聚焦于一门课程在一个学期内的完整教学过程，帮助教师全面掌握所带班级的学生学习状况，同时为学生提供个人学习分析与智能练习支持。 

系统采用 **系统管理员 → 教师（含助教） → 学生** 的两级管理模型： 

- **系统管理员** 仅管理教师账号，不接触任何教学业务数据。 

- **教师** 全权管理自己所带课程、班级和学生数据，可导入学生名单、各类教学数据，查看班级整体与学生个人画像，进行 AI 智能组卷与考试管理，向学生发送通知。 

- **助教** 由教师分配权限，协助教师管理指定班级或部分功能。 

- **学生** 登录后查看个人学习画像、成绩趋势、错题本，参加在线考试，使用 AI 学习助手。 

系统核心能力包括：多源教学数据导入与聚合、AI 智能组卷与出题、学生知识掌握度分析、学业预警与学生画像、错题分析与相似题目推荐、AI 对话式学习助手。评教功能不在本系统范围内。

## 2. 总体描述 

### 2.1 项目前景 

当前高校教师普遍面临以下痛点：

- 作业、考勤、实验、测验等数据分散在不同平台，难以汇聚分析。

- 无法快速定位班级整体薄弱知识点和需要重点关注的学生。

- 出题组卷耗时费力，缺乏基于知识点的智能题目生成与错题推荐能力。

- 学生缺乏针对性的个人学习分析，不知道自己哪里薄弱、该怎么补。

本项目构建一个 **以教师为中心、课程为维度、AI 为核心驱动力的教学辅助系统** ，帮助教师一站式完成数据导入、学情分析、智能组卷、学生画像与预警通知，同时为每位学生提供个人学习分析、错题训练与 AI 辅导，实现“数据驱动教学、AI 辅助学习”。 

### 2.2 产品功能概览 

系统功能按优先级划分为 **核心功能（必须完成）和扩展功能（有余力则做）** ： 

|**优先级**|**模块**|**功能说明**|
|:---:|:---:|:---:|
|**核心**|教学数据导入与管理|教师导入课程学生名单、作业成绩、考勤记录、实验报告成绩、课堂测验成绩、期中/期末考试成绩等多源数据|
|**核心**|教师教学驾驶舱|以班级/课程为单位的综合数据看板，支持单班查看与多班对比（仅限本人所带班级）|
|**核心**|学生画像与预警|每位学生的多维度画像（考勤、作业、测验、实验、课堂参与），预警清单与重点关注标记|
|**核心**|AI 智能出题与组卷|根据知识点、题型、难度自动生成题目，教师审核后组卷发布|
|**核心**|在线考试与错题分析|学生在线作答，客观题自动批改，错题自动分析并推荐相似练习题|
|**核心**|AI 对话助手|教师端教学助手与学生端学习助手，基于本地大模型提供智能问答|
|**核心**|通知中心|教师向学生发送通知，系统自动触发学业预警通知|
|**扩展**|课程教学报告|课程结束后自动生成课程整体教学分析报告（Word/PDF）|
|**扩展**|课堂互动记录|导入课堂提问、讨论参与等数据，纳入学生画像|
### 2.3 用户特征 

|**用户角色**|**主要职责**|**典型操作**|**技术水平**|**使用频率**|
|:---:|:---:|:---:|:---:|:---:|
|**系统管理员**|系统基础维护，仅管理教师账号|创建/禁用教师账号、重置密码、配置系统参数|具备IT系统管理能力|极低|
|**教师**|课程教学数据管理、学情分析、组卷与学生管理|导入学生名单与成绩、查看看板与学生画像、AI出题组卷、发布考试、发送通知、重点关注学生|具备基本办公软件操作能力|日常教学使用|
|**助教**|协助教师管理指定班级或功能|批改作业/实验成绩录入、查看学生数据、协助组卷（权限由教师分配）|具备基本办公软件操作能力|日常教学使用|
|**学生**|查看个人学习数据，参加在线测试，使用AI助手|查看个人画像与成绩趋势、错题本练习、在线考试、AI学习助手对话|日常使用手机/电脑|常态化使用|
### 2.4 约束 

- **开发周期** ：总时长4周（7月6日—8月2日），核心功能优先，扩展功能视进度决定。 

- **团队规模** ：5人，角色固定，不允许跨角色兼任导致关键路径阻塞。 

- **技术栈** ：前端 Vue 3 + Element Plus；后端 Spring Boot + MyBatis-Plus；数据库 MySQL 8.0；图表 ECharts 5。 

- **AI 方案** ：采用本地部署大模型（如 ChatGLM、Qwen 等开源模型通过 Ollama 部署），用于智能出题、知识分析、对话助手等。若本地部署遇到困难，可暂时使用云端 API 作为过渡方案。 

- **无 GPU 约束** ：演示环境无 GPU 服务器，需选择 CPU 可运行的轻量模型或量化版本，或使用云端 API 演示。 

- **数据来源** ：不接入真实教务系统，所有数据通过教师上传 Excel 文件导入。 

- **部署形态** ：Web 端响应式适配，不开发移动端 APP。 

- **权限模型** ：两级管理——系统管理员管理教师，教师管理学生。教师只能看到自己班级的数据，不能查看其他教师的班级。 

### 2.5 依赖与假设 

**外部依赖** ： 

- 本地部署的大模型服务稳定可用，推理延迟在可接受范围（CPU 环境单次推理≤60秒）。

- ECharts 5 及前端图表库的 CDN 或 npm 包稳定可用。

**假设条件** ： 

- 教师提供的导入文件符合预设模板格式。

- 数据量级：单门课程 1-3 个班级，每班 30-60 名学生，单学期数据条目在千条级别。

- 演示环境预置模拟历史数据以展示趋势分析与预警。

- 站内通知为系统内实现，不对接真实邮件/短信。

## 3. 需求描述 

### 3.1 对外接口描述 

#### 3.1.1 用户界面

系统提供基于 Web 浏览器的图形用户界面，采用 Vue 3 + Element Plus 构建，支持 Chrome、Edge 等主流浏览器。

**通用界面** ： 

- **登录页** ：所有角色共用统一入口，输入账号密码后系统自动识别角色并跳转至对应首页。首次登录强制修改初始密码。 

**系统管理员界面** ： 

- 登录后进入 **教师账号管理页（首页）** ，左侧菜单包含：教师账号管理、系统参数配置。界面风格简洁，不包含任何教学业务入口。 

**教师界面** ： 

- 登录后进入 **教学驾驶舱（首页）** ，默认展示最近操作班级的学情概览。 

- 左侧菜单包含：教学驾驶舱、班级管理、数据导入、学生画像、AI 出题组卷、考试管理、题库管理、通知中心。

- 顶部提供班级切换器，可选择单个班级或“多班合并视图”。

- 右下角悬浮 **AI 教学助手气泡** 。 

**助教界面** ： 

- 登录后进入 **教学驾驶舱（首页）** ，但可见班级和功能范围由分配教师限制。 

- 左侧菜单与教师类似，但部分功能（如组卷发布）可能为只读或无权限。

**学生界面** ： 

- 登录后进入 **个人学习中心（首页）** 。 

- 顶部导航：我的画像、成绩趋势、错题本、在线考试、AI 学习助手。

- 右下角悬浮 **AI 学习助手气泡** 。 

所有界面均提供操作提示与加载状态，支持主流浏览器。

#### 3.1.2 硬件接口

本系统为纯软件系统，无任何直接硬件接口。

#### 3.1.3 软件接口

|**接口名称**|**说明**|
|:---:|:---:|
|**后端 RESTful API**|前端通过 Axios 发送 HTTP 请求至 Spring Boot 后端，数据格式为 JSON。统一返回结构{ code, message, data }。提供用户认证、数据导入导出、考试管理、AI 调用等全部接口。|
|**本地大模型 API**|后端通过 RestTemplate 调用本地部署的大模型服务（如 Ollama 提供的 RESTful 接口）。请求格式：POST，携带 Prompt 及模型参数；响应返回 JSON，包含生成文本。需处理超时与重试。|
|**文档解析库**|Excel 解析使用 EasyExcel；Word 解析使用 Apache POI。将文件内容映射为 Java 实体后写入数据库。|
|**图表渲染库**|ECharts 5 用于所有图表渲染（柱状图、折线图、雷达图、饼图、仪表盘等）。|
#### 3.1.4 通信接口

- **前后端通信** ：基于 HTTP/1.1 协议，开发环境通过 Vite 代理解决跨域，生产环境通过 Nginx 反向代理或后端 CORS 配置。 

- **本地大模型调用** ：后端通过内网 HTTP 协议调用本地大模型服务（如http://localhost:11434/api/generate），无需加密传输。 

### 3.2 功能需求（按角色面板详细描述） 

#### 3.2.1 系统管理员功能面板

##### 面板1：教师账号管理（首页）

**功能** ：管理所有教师账号，不接触学生账号和教学数据。 

**页面布局** ： 

- 顶部：搜索栏（支持按姓名、工号、账号搜索）。

- 主体：教师账号列表表格。

**表格字段** ： 

|列名|说明|
|:---|:---|
|工号|教师工号，唯一标识|
|姓名|教师姓名|
|账号|登录账号（默认与工号相同）|
|状态|正常 / 已禁用|
|创建时间|账号创建日期|
|最后登录时间|最近一次登录时间|
|操作|禁用/启用、重置密码、编辑|
**操作功能** ： 

- **手动添加教师** ：点击“添加教师”按钮，弹出表单，填写工号、姓名、初始密码（必填），保存后账号自动生成。 

- **批量导入教师** ：点击“导入教师”，下载专用 Excel 模板（工号、姓名、初始密码），上传后系统批量创建账号。导入结果展示成功/失败条数，失败记录可下载错误日志。 

- **禁用/启用账号** ：点击操作列按钮，弹窗确认后执行。禁用后该教师无法登录。 

- **重置密码** ：点击后弹出新密码设置框，或自动生成随机密码并展示。 

- **编辑** ：可修改教师姓名，工号不可修改。 

- **搜索** ：按姓名或工号模糊搜索。 

##### 面板2：系统参数配置

|参数名称|说明|默认值|
|:---|:---|:---|
|大模型API地址|本地大模型服务的URL|[http://localhost:11434](http://localhost:11434/) |
|大模型名称|使用的模型名称|qwen2.5:7b|
|模型超时时间|单次推理最大等待时间（秒）|60|
|默认初始密码|批量导入教师时的默认密码|123456|
|预警规则配置|缺勤次数阈值、成绩下滑幅度阈值等|缺勤≥3次 / 连续两次成绩下降≥20%|
#### 3.2.2 教师功能面板

##### 面板1：教学驾驶舱（首页）

**功能** ：展示当前选中班级或课程的综合学情数据一览。 

**页面布局** ： 

- 顶部：班级切换器（下拉选择单个班级，或选择“多班合并视图”）。

- 第一行：统计卡片行（4-5个卡片）。

- 第二行：图表区（左图表 + 右图表）。

- 第三行：预警学生列表。

|卡片名称|内容|
|:---|:---|
|班级人数|当前班级学生总数|
|平均成绩|最近一次考试/测验班级平均分|
|出勤率|本学期平均出勤率（百分比）|
|作业提交率|本学期作业按时提交比例（百分比）|
|预警人数|当前处于预警状态的学生数量（红色高亮，可点击跳转）|
**图表区** ： 

- **班级成绩分布图** ：直方图或箱线图，展示最近一次考试成绩分布（横轴分数段，纵轴人数）。 

- **成绩趋势图** ：折线图，展示本学期各次测验/考试班级平均分变化趋势。 

- **考勤统计图** ：饼图或柱状图，展示出勤/迟到/请假/缺勤比例。 

- **作业提交统计图** ：堆叠柱状图，展示每次作业的按时提交、迟交、未交人数。 

- **知识点掌握度雷达图** ：基于考试错题统计，按章节/知识点展示班级平均掌握度。 

**预警学生列表** ： 

- 表格形式，字段：学号、姓名、预警类型（成绩下滑/缺勤过多/作业连续未交）、预警时间、严重程度（高/中/低）。

- 每行可点击进入该学生的详细画像页。

- 支持按预警类型筛选。

**交互** ：所有图表支持鼠标悬停显示详细数值，点击学生姓名可跳转至学生画像详情页。 

##### 面板2：班级管理

**功能** ：管理教师所带班级及学生名单。 

**班级列表页** ： 

- 显示教师所带的所有班级卡片，每个卡片展示班级名称、人数、本学期课程名。

- 支持新增班级（填写班级名称、课程名称、学期）。

- 点击班级卡片进入该班级的学生名单页。

**学生名单页** ： 

- 表格展示班级内所有学生：学号、姓名、性别（可选）、已导入数据类型标签。

- **导入学生名单** ：点击按钮，上传 Excel 文件（模板：学号、姓名、班级）。导入时，学号与系统中已有学生匹配则关联，不存在则自动创建学生账号并分配初始密码。

### 3.3 非功能性需求 

#### 3.3.1 安全性

- **认证与授权** ：所有用户通过 JWT Token 鉴权，后端接口根据角色进行权限校验。教师仅可访问自己班级数据，学生仅可访问本人数据。 

- **数据隔离** ：后端查询必须基于当前用户 ID 和角色进行数据过滤，不允许通过修改请求参数越权访问。 

- **文件上传安全** ：校验文件后缀白名单（.xlsx, .xls），限制文件大小（≤10MB），后端进行魔数校验。 

- **敏感信息保护** ：本地大模型 API 地址、数据库密码等配置不硬编码，存放于配置文件并加入 .gitignore；用户密码加密存储。 

- **防注入** ：使用 MyBatis-Plus 预编译语句，避免 SQL 拼接；前端输入进行 XSS 过滤。 

#### 3.3.2 性能

- **页面加载** ：首页及主要页面≤2秒，看板图表渲染≤3秒。 

- **数据导入** ：单次上传包含 50 条记录的 Excel 文件，解析入库及反馈≤3秒。 

- **AI 推理** ：本地大模型单次推理≤30秒（有 GPU）或 ≤60秒（纯 CPU），需在前端显示加载状态。 

- **并发** ：支持 5 名教师同时操作，50 名学生同时在线考试。 

- **内存占用** ：JVM 堆内存≤1GB，本地大模型独立进程资源另计。 

#### 3.3.3 软件质量要求

. **易用性** ： 

  o. 教师端功能入口清晰，关键操作（如发布考试、删除学生）有二次确认。

  o. 学生端界面简洁，移动端响应式适配良好。

  o. 数据导入失败时明确提示错误行号和原因，提供模板下载。

  o. 所有图表支持鼠标悬停交互。

. **可维护性** ： 

  o. 后端按 controller/service/mapper 分层，命名规范，关键逻辑加注释。

  o. 前端组件化开发，图表配置封装为独立组件，API 请求统一封装。

  o. README 包含环境搭建、启动步骤、本地大模型部署指南。

. **可扩展性** ： 

  o. AI 模块预留接口，可切换不同大模型。

  o. 数据类型（作业/考勤/实验等）通过配置扩展，便于新增。

  o. 预警规则通过数据库或配置文件管理，支持动态调整阈值。

#### 3.3.4 通知系统非功能要求

- **通知延迟** ：手动发送通知即时送达；系统自动预警在触发条件满足后 10 秒内生成通知。 

- **通知去重** ：同一学生的同类预警通知在 24 小时内不重复生成。 

- **已读管理** ：支持标记已读/未读，未读通知红点数字展示。 

#### 3.3.5 日志系统非功能要求

- **日志记录** ：采用异步方式记录，不影响主业务流程性能。 

- **日志类型** ：操作日志（用户登录、数据导入、考试发布等）、AI 调用日志（请求时间、耗时、成功/失败）、错误日志。 

- **日志保留** ：至少保留 30 天，支持按需清理。
